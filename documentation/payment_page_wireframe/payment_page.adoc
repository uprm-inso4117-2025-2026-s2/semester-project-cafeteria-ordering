:toc:
:toclevels: 2

== Objective ==
Define the visual structure and layout of the Payment Page within the Cafeteria Ordering System mobile ordering flow.

This document describes the wireframe design and component placement that will guide future UI design and implementation.

== Wireframe Overview ==
The Payment Page wireframe represents the final checkout screen where users review their order and proceed with payment.

The purpose of this wireframe is to:

- Establish layout structure
- Define section grouping
- Clarify visual priority of elements
- Ensure a clean and mobile-friendly design
- Serve as a reference for UI implementation

== Wireframe Structure ==
The Payment Page wireframe is divided into the following primary sections:

=== 1. Return Button Section ===

Position: 

- Top left of the screen

Includes:

- Button allowing the option to continue modifying the order

=== 2. Order Summary Section ===

Position:

- Top portion of the screen

Includes:

- List of selected items
- Quantity displayed before item name
- Total price per item aligned to the right
- Individual unit price displayed beneath item name in smaller text
- Subtotal, taxes, and final total displayed at the bottom of the section

=== 3. Payment Method Section ===

Position:

- Below Order Summary

Include:

- Option to select a payment method
- Option to add a new payment method
- Display area for saved payment methods

=== 4. New Payment Entry Form (Conditional) ===

Position:

- Appears below payment method selection when "Add New Card" is selected

Includes:

- Card number field
- Expiration date field
- CVV field
- Cardholder name field
- Country field
- Address Line field
- City field
- State/Territory field
- Postal Code field

=== 5. Payment Section ===

Position:

- Bottom of the screen

Includes:

- Primary "Pay Now" button
