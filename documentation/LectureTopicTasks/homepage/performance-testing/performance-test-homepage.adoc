# Homepage Load & Performance Testing

## Test Setup
- Increased the dummy dataset from 5 menu items to 100 menu items to simulate higher frontend load  

## Load Simulation Strategy
Simulated load by interacting with the app continuously by:

  - Rapid scrolling through the menu item list  
  - Switching between category filters multiple times  
  - Performing multiple search queries in sequence (for example: "latte", "burger")  
  - Opening item details repeatedly  

## Alignment with Performance Requirements
This test checks how the system behaves under increased data and repeated interactions by focusing on:

- Menu loading responsiveness with more data  
- Search response time during repeated use  
- UI stability during continuous interaction  
- Responsiveness of category filtering  

## Observations
- A slight delay (initial render time increase) was noticed when loading the larger dataset  
- After the initial load, scrolling remained smooth  
- Switching between categories was responsive and consistent  
- Search functionality worked without noticeable delay  
- Opening item details was responsive  
- No crashes or major UI issues were observed  

## Conclusion
The homepage handled the increased dataset well. While there was a small delay during the initial load, all other interactions remained smooth and consistent.
