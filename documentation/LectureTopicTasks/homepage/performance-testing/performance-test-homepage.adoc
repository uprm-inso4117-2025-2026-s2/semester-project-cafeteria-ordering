# Homepage Performance Testing

## Test Setup
- Increased the dummy dataset from 5 items to 100 items and tested the homepage with the larger dataset

## Actions Performed
- I scrolled through the menu item list
- Switched between category filters (Breakfast, Lunch, Rating)
- Used the search bar to filter items by name like "latte" and "Burger"
- Opened item details from the homepage

## Observations
- A slight delay was noticed during the initial load when rendering the larger dataset
- After loading however, scrolling remained smooth
- Switching between categories was responsive and consistent
- Search functionality worked correctly without noticeable delay (It was more responsive than expected)
- No crashes or major UI issues were observed

## Conclusion
The homepage handled the increased dataset well. While there was a small delay during the initial load, overall performance remained stable and interactions continued to function as expected.
