== Description
All 19 test cases passed successfully (19/19). The password reset and recovery flow implementation was verified across the following areas:

* *Step 1 - Forgot Password UI (2 tests):* The Forgot Password screen renders correctly with the "Forgot Password" title, email field label, and "Send recovery link" button.
* *Step 2 - Invalid Email Submissions (2 tests):* Empty email submission shows "Email is required." error. Invalid email format ("notanemail") shows "Please enter a valid email address." error. Both block form submission.
* *Step 3 - Unregistered Email (1 test):* Submitting an unregistered email ("ghost@uni.edu") shows a generic success popup ("Check your email"), preventing email enumeration.
* *Step 4 - Valid Email Reset Request (4 tests):* Submitting a valid email triggers the reset request, displays the success popup with "Password reset link has been sent" message, shows "Return to Login" button that navigates to /login, and displays an error message when the backend request fails.
* *Step 5 - Reset Password UI (4 tests):* The Reset Password screen renders with the "Reset Password" title, "New Password" and "Confirm Password" fields, a password requirements checklist (8 characters, 1 uppercase, 1 lowercase, 1 number), and the "Reset password" button.
* *Step 6 - Password Validations (3 tests):* Short password ("short") shows "Password must be at least 8 characters." error. Mismatched passwords show "Passwords don't match" error. Empty password shows "Password is required." error. All block form submission.
* *Step 7 - Valid Password Submission (3 tests):* Matching valid passwords ("SecureReset2026!") trigger the update call and show the "Password reset complete" success popup. Backend failures display an appropriate error message. The "Back to sign up" button navigates to /signup.
