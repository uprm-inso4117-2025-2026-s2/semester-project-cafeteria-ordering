= Testing Supabase Signup Backend
:toc:
:icons: font

This test validates backend signup behavior independently from the frontend signup screen.

== Quick Run

[source,bash]
----
npm run test:signup-backend
----

This script validates:

* New account signup with generated unique email
* Phone metadata persistence to profile (when session/profile read is available)
* Duplicate signup handling behavior
* Weak password rejection
* Profile visibility check when session is available

== Prerequisites

. Ensure `.env` has required values:
+
[source,bash]
----
EXPO_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
EXPO_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here
SUPABASE_TEST_SIGNUP_PASSWORD=StrongPassword123
----

. Ensure database trigger for profile creation is active.

== Notes

When email confirmation is enabled, signup may return no active session. In that mode, profile visibility check is skipped and should be validated from Supabase logs/tables.
